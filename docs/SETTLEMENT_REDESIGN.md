## 数据层改造

### rites.json 新增字段

```json
{
  "id": 1,
  "name": "探索废弃神殿",
  "stages": [
    {
      "text": "神殿入口笼罩着诡异的雾气，你握紧武器向前迈进。",
      "check": { "type": "solo", "attribute": "phy", "required_successes": 3 },
      "success_text": "你屏住呼吸穿过雾气，前方豁然开朗。",
      "failure_text": "迷雾中传来低沉的咆哮——你被发现了！",
```

## Phase 1：UI 框架（先跑起来）

| 步骤 | 内容 | 估时 |
|------|------|------|
| 1.1 | 创建 `SettlementScreen` — 全屏或大半屏弹窗，左右 HSplitContainer | 30min |
| 1.2 | 左侧骰子面板 `DicePanel` — 展示角色卡(名称+头像+属性)、待投骰子区域 | 30min |
| 1.3 | 右侧叙事面板 `NarrativePanel` — 长文本区(ScrollContainer) + 状态指示器 | 20min |

## Phase 2：骰子动画

| 步骤 | 内容 | 估时 |
|------|------|------|
| 2.1 | `DieNode` — 单颗骰子节点，带翻滚动画(Tween + random frame) | 40min |
| 2.2 | `DiceTray` — 骰子盘，管理 N 颗骰子的投掷、落定动画 | 30min |
| 2.3 | 成功/失败高亮 — 绿色(✅)、红色(❌)、金色(金骰子) | 20min |

## Phase 3：分步叙事引擎

| 步骤 | 内容 | 估时 |
|------|------|------|
| 3.1 | `StageManager` — 管理 stages 数组的逐步推进 | 30min |
| 3.2 | 打字机效果 — 叙事文本逐字显示，可点按加速 | 20min |
| 3.3 | 检定触发 — 每个 stage 完成后投骰子→显示结果→进入下一 stage | 20min |
| 3.4 | 失败处理 — 根据 `on_failure` 决定继续/跳转/结束 | 20min |

## Phase 4：与现有系统对接

| 步骤 | 内容 | 估时 |
|------|------|------|
| 4.1 | `_settle_next` 改为调用 SettlementScreen | 15min |
| 4.2 | rites.json 补全 stages 字段（MVP 5个仪式） | 30min |
| 4.3 | 兼容旧数据 — 无 stages 的仪式自动生成单 stage | 10min |

## 不做的事

- 不做真实的 3D 骰子物理（Tween 模拟已足够）
- 不做语音/音效（Phase 4 可选）
- 不改造 `SettlementPopup.gd`（直接替换为新系统）

## 验收标准

- 至少 1 个仪式有完整的 2+ stages 叙事流
- 骰子有翻滚→落定→高亮的动画链
- 文本有打字机效果，可点按跳过
- 失败有视觉反馈（红色闪烁/震动），不会卡住流程
