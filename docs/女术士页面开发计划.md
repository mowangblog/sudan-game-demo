## 原版体验调研

### 核心机制

原版中，**女术士是一个独立的NPC页面**，玩家与女术士对话来管理苏丹卡。这个页面承担了以下职责：

| 功能 | 原版交互 | 我们现状 |
|------|----------|----------|
| **抽苏丹卡** | 点击牌盒 → 女术士台词 → 卡牌出现 + 品级讲解 | `GameManager._draw_sultan_card()` 直接静默入手牌 |
| **折卡仪式** | 女术士"用力折断刚刚被完成的苏丹卡" → 抽新卡 | `consume_sultan_card()` 静默销毁，第二天自动补抽 |
| **换卡** | 7日内有N次机会换一张随机卡（倒计时不变） | 不存在 |
| **进度面板** | 已存活X天、折断了哪些卡、卡池剩余X/28 | 不存在 |
| **NPC对话** | 女术士有独特语气（嘲讽+恭顺）、按卡类型差分台词 | 不存在 |
| **牌盒视觉** | 神秘木匣/宝盒，交互式点击 | 不存在 |

### 关键规则

1. **抽卡时机**：苏丹卡被折断后才抽下一张；嵌入仪式中的卡即使倒计时归零也不触发处刑/抽新卡
2. **卡池构成**：4类型×4等级 = 28张（石2×4=8, 铜2×4=8, 银2×4=8, 金1×4=4）
3. **换卡规则**：换卡次数由难度设定（默认7次），倒计时不变；命运商店可额外购买5次
4. **品级讲解**：首次抽卡时女术士会逐级解释品级概念

### 仪式感要素（"缺了一点感觉"的根源）

原版的核心仪式感由3个环节串联：

```
折断旧卡 → 女术士捧起牌盒 → 点击牌盒抽新卡 → 女术士品级讲解台词
```

我们目前直接静默抽卡入手牌，完全跳过了这3个环节。

## 开发计划（5 Phase）

### Phase 1: 女术士对话页面 — SorceressScene

**目标**：创建独立的女术士NPC对话页面，玩家点击地图上的女术士图标进入

**内容**：
- 女术士NPC形象（半遮挡面容的神秘女性，紫色魔法迷雾）
- 对话界面：左侧女术士立绘，右侧对话框（打字机效果）
- 4个对话入口：
  - 「抽取摄政王令」→ 进牌盒抽卡流程（Phase 2）
  - 「交换摄政王令」→ 换卡流程（Phase 3）
  - 「查看进度」→ 进度面板（Phase 4）
  - 「与她聊聊」→ 静态对话树（规则讲解/世界观）
- 入口触发：地图上放置女术士图标节点，点击进入

**文件**：
- `scripts/ui/SorceressScene.gd` — 页面主控
- `scripts/ui/SorceressDialogue.gd` — 对话树数据+渲染
- 复用 `PopupManager` 的弹窗基础设施

**数据**：
- `data/sorceress_dialogues.json` — 女术士台词库（按卡类型/品级差分）

**触发时机**：
- 开局 → 强制进入女术士页面抽第一张卡（替代当前 `start_game` 的静默抽卡）
- 折断苏丹卡后 → 弹出女术士页面抽新卡（替代当前 `_on_day_started` 的静默补抽）

---

### Phase 2: 牌盒抽卡交互 — CardBox

**目标**：牌盒点击→抽卡动画→品级讲解→卡牌入手牌

**交互流程**：
```
女术士："请点击牌盒，抽取您的卡牌吧，阁下。"
        ↓
    牌盒视觉（木匣/宝盒，居中，可点击）
        ↓ 玩家点击
    牌盒打开动画（翻盖/光效/粒子）
        ↓ 0.5s
    卡牌从牌盒飞出 → 屏幕中央展示 2s
        ↓
    女术士品级讲解台词：
    "一张黄金杀戮卡！请注意...黄金是最高品级..."
        ↓ 打字机效果完成
    「我知道了」按钮 → 卡牌滑入手牌 → 关闭页面
```

**视觉**：
- 牌盒：深色木纹底+金色边框的Panel，居中 120×80
- 打开动画：缩放+旋转+光效（Tween）
- 卡牌展示：全屏半透明遮罩+卡牌居中大图（用现有CardFactory样式放大3倍）
- 品级讲解：跟卡类型/品级差分的台词（Phase 1 的 sorceress_dialogues.json）

**文件**：
- `scripts/ui/CardBox.gd` — 牌盒交互组件
- 动画用 Tween（不需要shader）

**首次抽卡特殊处理**：
- 第一次进入时，女术士先讲解4种卡类型概念
- 然后才出现牌盒

**非首次抽卡差分**：
- 女术士台词："请点击牌盒，抽取您的下一张卡牌吧，阁下。"
- 不再重复讲解品级概念

---

### Phase 3: 换卡功能

**目标**：允许玩家将手上的苏丹卡还给女术士，重新抽一张随机卡

**规则**：
- 每局初始换卡次数 = 7（对应原版默认难度）
- 换卡后倒计时不变（不重置7天）
- 换出的卡回到卡池，从卡池随机抽一张新卡
- 如果卡池空了，无法换卡

**交互流程**：
```
女术士："这7日你有N次机会可以把手上的一张苏丹卡还给我..."
        ↓
    手牌上的苏丹卡自动展示（不可拖拽，仅展示）
        ↓ 玩家点击苏丹卡
    女术士确认："您确定要将这张卡放回宝盒吗？"
        ↓ 确认
    旧卡折断动画 → 牌盒抽卡流程（Phase 2）
        ↓
    新卡入手牌，倒计时不变
```

**数据改动**：
- `GameManager` 新增 `swap_tokens: int = 7`
- `DataManager` 新增 `return_sultan_card(card_id)` → 把卡放回卡池
- `GameManager` 新增 `swap_sultan_card()` → 换卡逻辑

### Phase 4: 进度面板

**目标**：在女术士页面展示折卡进度/存活天数/卡池状态

**布局**：
```
╔════════════════════════════════╗
║  已存活 23 天    折卡 6/28     ║
║                                ║
║  ── 已折断 ──                  ║
║  石欢愉 ✓  铜征伐 ✓  ...      ║
║                                ║
║  ── 卡池剩余 ──                ║
║  欢愉: 石0 铜1 银2 金1         ║
║  奢靡: 石2 铜1 银0 金1         ║
║  征伐: 石1 铜0 银1 金0         ║
║  杀戮: 石1 铜0 银0 金1         ║
║                                ║
║  ── 当前手牌 ──                ║
║  铜欢愉 (剩余5天)              ║
╚════════════════════════════════╝
```

**数据改动**：
- `DataManager` 新增 `get_card_pool_stats()` → 返回各类型剩余数量
- `GameManager` 新增 `consumed_cards: Array` → 已折断卡记录
- `GameManager` 新增 `days_alive: int` → 存活天数（TurnManager.day_number）

### Phase 5: 折卡仪式

**目标**：苏丹卡被成功折断时，展示女术士折卡动画而非静默销毁

**触发时机**：
- 仪式结算成功且消耗了苏丹卡 → `consume_sultan_card()` 被调用时

**交互流程**：
```
结算页显示成功 → 短暂停顿 0.5s
        ↓
    女术士折卡动画：
    - 女术士头像闪现（半屏）
    - 台词："（她用力折断了刚刚被完成的苏丹卡）"
    - 折断动画：卡牌居中 → 女术士双手→咔嚓断裂 → 碎片飘散
        ↓ 1.5s
    「抽取下一张」按钮 → 进牌盒抽卡流程（Phase 2）
    或 「稍后再来」按钮 → 返回主界面，下次主动进入女术士页面
```

**注意事项**：
- 折卡+抽新卡流程不应阻塞回合结算——折卡动画可以叠加在结算流程末尾
- 如果玩家选择「稍后再来」，则 `active_sultan_card` 暂时为空，玩家可以自由行动直到主动找女术士抽卡
- 这也还原了原版的"拖延策略"：折断后不立即抽新卡可以多几天自由行动

**文件**：
- `scripts/ui/CardBreakAnimation.gd` — 折卡视觉组件

## Phase 依赖关系

```
Phase 1 (女术士页面) ──→ Phase 2 (牌盒抽卡) ──→ Phase 3 (换卡)
                    │                          ──→ Phase 4 (进度面板)
                    └──→ Phase 5 (折卡仪式)
```

- Phase 1 是所有后续的基础
- Phase 2 必须跟 Phase 1 一起完成（抽卡是核心流程）
- Phase 3/4/5 可以独立迭代，优先级按编号

## 实施建议

1. **Phase 1+2 一起做**，因为抽卡流程是女术士页面的核心交互，拆开做会导致流程断裂
2. Phase 1+2 完成后，`GameManager._draw_sultan_card()` 改为触发女术士页面而非静默抽卡
3. Phase 5 的折卡动画需要修改 `consume_sultan_card()` 的调用链——结算成功后不再静默清空，而是触发折卡流程
4. Phase 5 完成后，原版的完整"折断→抽新卡"循环才算闭环

## MVP 范围建议

**必须做**：Phase 1 + Phase 2（女术士页面+牌盒抽卡）— 恢复原版的仪式感
**建议做**：Phase 5（折卡仪式）— 完善消耗闭环
**可选**：Phase 3（换卡）、Phase 4（进度面板）— 功能增值，不影响核心循环
