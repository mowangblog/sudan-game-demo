# 女术士页面开发计划

> 基于原版《苏丹的游戏》BWIKI 资料整理（以下调研内容保留原版术语以对照）
> 参考：[牌盒](https://wiki.biligame.com/sultansgame/牌盒) · [女术士](https://wiki.biligame.com/sultansgame/女术士)
>
> **本地化术语对照**：苏丹卡→摄政王令 / 牌盒→令匣 / 折卡→折令 / 抽苏丹卡→抽摄政王令

## 原版体验调研

### 核心机制

原版中，**女术士是一个独立的NPC页面**，玩家与女术士对话来管理苏丹卡（→摄政王令）。这个页面承担了以下职责：

| 功能 | 原版交互 | 我们现状 |
|------|----------|----------|
| **抽摄政王令** | 点击令匣 → 女术士台词 → 卡牌出现 + 品级讲解 | `GameManager._draw_sultan_card()` 直接静默入手牌 |
| **折令仪式** | 女术士"用力折断刚刚被完成的摄政王令" → 抽新令 | `consume_sultan_card()` 静默销毁，第二天自动补抽 |
| **换令** | 7日内有N次机会换一张随机令（倒计时不变） | 不存在 |
| **进度面板** | 已存活X天、折断了哪些令、令池剩余X/28 | 不存在 |
| **NPC对话** | 女术士有独特语气（嘲讽+恭顺）、按令类型差分台词 | 不存在 |
| **令匣视觉** | 神秘木匣/宝盒，交互式点击 | 不存在 |

### 关键规则

1. **抽令时机**：摄政王令被折断后才抽下一张；嵌入仪式中的令即使倒计时归零也不触发处刑/抽新令
2. **令池构成**：4类型×4等级 = 28令（石2×4=8, 铜2×4=8, 银2×4=8, 金1×4=4）
3. **换令规则**：换令次数由难度设定（默认7次），倒计时不变；命运商店可额外购买5次
4. **品级讲解**：首次抽令时女术士会逐级解释品级概念

### 仪式感要素（"缺了一点感觉"的根源）

原版的核心仪式感由3个环节串联：

```
折断旧令 → 女术士捧起令匣 → 点击令匣抽新令 → 女术士品级讲解台词
```

我们目前直接静默抽令入手牌，完全跳过了这3个环节。

## 开发计划（5 Phase）

### Phase 1: 女术士对话页面 — SorceressScene

**目标**：创建独立的女术士NPC对话页面，玩家点击地图上的女术士图标进入

**内容**：
- 女术士NPC形象（半遮挡面容的神秘女性，紫色魔法迷雾）
- 对话界面：左侧女术士立绘，右侧对话框（打字机效果）
- 4个对话入口：
  - 「抽取摄政王令」→ 进令匣抽令流程（Phase 2）
  - 「交换摄政王令」→ 换令流程（Phase 3）
  - 「查看进度」→ 进度面板（Phase 4）
  - 「与她聊聊」→ 静态对话树（规则讲解/世界观）
- 入口触发：地图上放置女术士图标节点，点击进入

**文件**：
- `scripts/ui/SorceressScene.gd` — 页面主控
- `scripts/ui/SorceressDialogue.gd` — 对话树数据+渲染
- 复用 `PopupManager` 的弹窗基础设施

**数据**：
- `data/sorceress_dialogues.json` — 女术士台词库（按令类型/品级差分）

**触发时机**：
- 开局 → 强制进入女术士页面抽第一张令（替代当前 `start_game` 的静默抽令）
- 折断摄政王令后 → 弹出女术士页面抽新令（替代当前 `_on_day_started` 的静默补抽）

---

### Phase 2: 令匣抽令交互 — CardBox

**目标**：令匣点击→抽令动画→品级讲解→令牌入手牌

**交互流程**：
```
女术士："请点击令匣，抽取您的摄政王令吧，阁下。"
        ↓
    令匣视觉（深色木匣+金纹，居中，可点击）
        ↓ 玩家点击
    令匣打开动画（翻盖/光效/粒子）
        ↓ 0.5s
    令牌从令匣飞出 → 屏幕中央展示 2s
        ↓
    女术士品级讲解台词：
    "一张黄金杀戮令！请注意...黄金是最高品级..."
        ↓ 打字机效果完成
    「我知道了」按钮 → 令牌滑入手牌 → 关闭页面
```

**视觉**：
- 令匣：深色木纹底+金色边框的Panel，居中 120×80
- 打开动画：缩放+旋转+光效（Tween）
- 令牌展示：全屏半透明遮罩+令牌居中大图（用现有CardFactory样式放大3倍）
- 品级讲解：跟令类型/品级差分的台词（Phase 1 的 sorceress_dialogues.json）

**文件**：
- `scripts/ui/CardBox.gd` — 令匣交互组件
- 动画用 Tween（不需要shader）

**首次抽令特殊处理**：
- 第一次进入时，女术士先讲解4种令类型概念（欢愉/奢靡/征伐/杀戮）
- 然后才出现令匣

**非首次抽令差分**：
- 女术士台词："请点击令匣，抽取您的下一张摄政王令吧，阁下。"
- 不再重复讲解品级概念

---

### Phase 3: 换令功能

**目标**：允许玩家将手上的摄政王令还给女术士，重新抽一张随机令

**规则**：
- 每局初始换令次数 = 7（对应原版默认难度）
- 换令后倒计时不变（不重置7天）
- 换出的令回到令池，从令池随机抽一张新令
- 如果令池空了，无法换令

**交互流程**：
```
女术士："这7日你有N次机会可以把手上的一张摄政王令还给我..."
        ↓
    手牌上的摄政王令自动展示（不可拖拽，仅展示）
        ↓ 玩家点击摄政王令
    女术士确认："您确定要将此令放回令匣吗？"
        ↓ 确认
    旧令折断动画 → 令匣抽令流程（Phase 2）
        ↓
    新令入手牌，倒计时不变
```

**数据改动**：
- `GameManager` 新增 `swap_tokens: int = 7`
- `DataManager` 新增 `return_sultan_card(card_id)` → 把令放回令池
- `GameManager` 新增 `swap_sultan_card()` → 换令逻辑

---

### Phase 4: 进度面板

**目标**：在女术士页面展示折令进度/存活天数/令池状态

**布局**：
```
╔════════════════════════════════╗
║  已存活 23 天    折令 6/28     ║
║                                ║
║  ── 已折断 ──                  ║
║  石欢愉 ✓  铜征伐 ✓  ...      ║
║                                ║
║  ── 令池剩余 ──                ║
║  欢愉: 石0 铜1 银2 金1         ║
║  奢靡: 石2 铜1 银0 金1         ║
║  征伐: 石1 铜0 银1 金0         ║
║  杀戮: 石1 铜0 银0 金1         ║
║                                ║
║  ── 当前手中 ──                ║
║  铜欢愉 (剩余5天)              ║
╚════════════════════════════════╝
```

**数据改动**：
- `DataManager` 新增 `get_card_pool_stats()` → 返回各类型剩余数量
- `GameManager` 新增 `consumed_cards: Array` → 已折断令记录
- `GameManager` 新增 `days_alive: int` → 存活天数（TurnManager.day_number）

---

### Phase 5: 折令仪式

**目标**：摄政王令被成功折断时，展示女术士折令动画而非静默销毁

**触发时机**：
- 仪式结算成功且消耗了摄政王令 → `consume_sultan_card()` 被调用时

**交互流程**：
```
结算页显示成功 → 短暂停顿 0.5s
        ↓
    女术士折令动画：
    - 女术士头像闪现（半屏）
    - 台词："（她用力折断了刚刚被完成的摄政王令）"
    - 折断动画：令牌居中 → 女术士双手→咔嚓断裂 → 碎片飘散
        ↓ 1.5s
    「抽取下一张」按钮 → 进令匣抽令流程（Phase 2）
    或 「稍后再来」按钮 → 返回主界面，下次主动进入女术士页面
```

**注意事项**：
- 折令+抽新令流程不应阻塞回合结算——折令动画可以叠加在结算流程末尾
- 如果玩家选择「稍后再来」，则 `active_sultan_card` 暂时为空，玩家可以自由行动直到主动找女术士抽令
- 这也还原了原版的"拖延策略"：折断后不立即抽新令可以多几天自由行动

**文件**：
- `scripts/ui/CardBreakAnimation.gd` — 折令视觉组件

## Phase 依赖关系

```
Phase 1 (女术士页面) ──→ Phase 2 (令匣抽令) ──→ Phase 3 (换令)
                    │                          ──→ Phase 4 (进度面板)
                    └──→ Phase 5 (折令仪式)
```

- Phase 1 是所有后续的基础
- Phase 2 必须跟 Phase 1 一起完成（抽令是核心流程）
- Phase 3/4/5 可以独立迭代，优先级按编号

## 实施建议

1. **Phase 1+2 一起做**，因为抽令流程是女术士页面的核心交互，拆开做会导致流程断裂
2. Phase 1+2 完成后，`GameManager._draw_sultan_card()` 改为触发女术士页面而非静默抽令
3. Phase 5 的折令动画需要修改 `consume_sultan_card()` 的调用链——结算成功后不再静默清空，而是触发折令流程
4. Phase 5 完成后，原版的完整"折断→抽新令"循环才算闭环

## MVP 范围建议

**必须做**：Phase 1 + Phase 2（女术士页面+令匣抽令）— 恢复原版的仪式感
**建议做**：Phase 5（折令仪式）— 完善消耗闭环
**可选**：Phase 3（换令）、Phase 4（进度面板）— 功能增值，不影响核心循环
