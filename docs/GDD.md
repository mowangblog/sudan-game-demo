## 1. 游戏概述与核心愿景

### 1.1 基本信息

| 项目 | 说明 |
|------|------|
| **游戏名称** | 苏丹的游戏（Sultan's Game） — 复刻学习版 |
| **原版开发商** | Double Cross / 2P Games |
| **复刻性质** | 非商业学习项目，用于美术与 UI 设计练习 |
| **游戏类型** | 卡牌驱动的回合制策略生存 RPG |
| **引擎** | Godot 4.6 |
| **目标平台** | PC（窗口化，1920×1080 基准分辨率） |
| **脚本语言** | GDScript |
| **目标受众** | 策略游戏爱好者、叙事游戏爱好者、原版玩家 |

### 1.2 核心体验一句话

> **"你被迫在7天内完成一张残酷的卡牌挑战——每一次选择都在改写你和身边人的命运。"**

### 1.3 玩家体验目标

| 体验维度 | 目标感受 | 设计手段 |
|----------|----------|----------|
| **紧迫感** | 倒计时的压迫感——7天期限如悬顶之剑 | 苏丹卡7天限时、仪式限时 |
| **道德困境** | 每个选择都有代价，没有"正确答案" | 多结局、角色关系变化、后果反馈 |
| **策略深度** | 资源有限，必须权衡取舍 | 多种卡牌处理路径、资源管理系统 |
| **叙事沉浸** | 一千零一夜式的异域故事感 | 角色卡牌化、事件叙事文本 |
| **重复可玩** | 每局不同，多周目有成长感 | 苏丹卡随机、积分解锁、多结局 |

### 1.4 与原版的关键差异声明

本复刻为**学习项目**，以下是与原版有意为之的差异：

| 差异项 | 原版 | 复刻版 | 原因 |
|--------|------|--------|------|
| 内容规模 | ~50+角色，~100+仪式/事件 | MVP约5角色，15仪式，20事件 | 学习项目范围限制 |
| 美术资产 | 完整原创美术 | 学习风格美术（简化或替代） | 美术学习练习 |
| 音频 | 完整音效+配乐 | 暂无（Phase 4可选） | 降低初期复杂度 |
| 创意工坊 | 支持 | 不支持 | 非学习目标 |
| 多语言 | 简中/繁中/英/日 | 简中 only | 降低工作量 |

> ⚠️ **假设标记**：我假设 MVP 的5角色/15仪式/20事件足以支撑完整的游戏循环体验。如果测试发现数量不足以产生紧迫感和策略深度，需要调整。请标记。

## 2. 核心循环与回合系统

### 2.1 宏观循环：周循环

```
┌──────────────────────────────────────────────┐
│            周循环 (Weekly Loop)               │
│                                              │
│  开始新一周                                   │
│     ↓                                        │
│  从牌盒抽取一张苏丹卡                          │
│     ↓                                        │
│  7天倒计时开始                                 │
│     ↓                                        │
│  ┌─── 每日循环 × 7 ───┐                      │
│  │  (详见 2.2)        │                      │
│  └────────────────────┘                      │
│     ↓                                        │
│  7天到期 → 检查苏丹卡是否已消耗                 │
│     ├─ 已消耗 → 进入下一周                     │
│     └─ 未消耗 → 触发死亡（头身分离术）          │
│                → 游戏结束                      │
└──────────────────────────────────────────────┘
```

### 2.2 微观循环：每日循环

| 阶段 | 玩家操作 | 系统行为 |
|------|----------|----------|
| **回合开始** | 无 | 弹出剧情事件；地图出现新仪式；手牌出现事件提示 |
| **玩家决策** | 选择事件选项；将卡牌拖入仪式配置；将卡牌拖入"俺寻思" | — |
| **俺寻思** | 核心探索机制——拖入卡牌触发探索事件/发现新仪式 | 详见 2.5 |
| **结算判定** | — | 仪式根据属性、卡牌、前置事件检定结果 |
| **点击「下一天」** | — | 所有仪式限时-1、用时-1；用时=0的仪式结算；进入新一天 |

### 2.3 每日结算优先级

> **原版规则**：游戏越早出现的仪式拥有越高优先级（按仪式ID从小到大）

| 顺序 | 结算阶段 | 说明 |
|------|----------|------|
| 1 | 倒计时减少 | 各卡牌倒计时-1 |
| 2 | 卡牌自带 vanish | 倒计时=0时触发的事件 |
| 3 | 仪式结算 | 按仪式ID排序，ID越小优先 |
| 4 | 卡牌自带 post_rite | 仪式使用后额外执行的动作 |

### 2.4 时间概念

| 术语 | 定义 |
|------|------|
| **回合 (round)** | 游戏中的一天，基本时间单位 |
| **周 (week)** | 7个回合，对应一张苏丹卡的处理期限 |
| **限时** | 仪式地图上显示的红色数字，表示该仪式剩余可用天数 |
| **用时** | 仪式详情右上角青色数字，表示该仪式执行所需天数 |

> `[PLACEHOLDER]`：我假设一周固定7天。原版是否有因剧情事件缩短或延长的情况？需验证。

### 2.5 "俺寻思"机制

> **核心探索入口**：这是原版中非常重要的机制，初稿仅一笔带过，现在补充完整。

"俺寻思"（Insight）是一个**特殊的拖放区域**，玩家可以将卡牌拖入其中来触发探索、发现、处理事件。

| 功能 | 说明 |
|------|------|
| **奢靡卡处理** | 将奢靡卡拖入 → 触发扩建（消耗3/5/10/20金） |
| **杀戮卡处理** | 将杀戮卡拖入 → 触发随机杀人事件 |
| **金币探索** | 将金币拖入 → 触发狩猎实战/黑吃黑（检定≥5可获权势+1） |
| **角色探索** | 将角色拖入 → 触发角色专属剧情线 |
| **道具探索** | 将特定道具拖入 → 发现隐藏事件 |

> **设计意图**：俺寻思是游戏的"万能探索器"——玩家不知道拖入什么会发生什么，这创造了探索发现的乐趣。**拖入后的结果应该有惊喜感**——有时候是好事，有时候是坏事，让玩家忍不住想尝试。

## 3. 苏丹卡与卡牌系统

### 3.1 苏丹卡分类

**四种类型 × 四个等级 = 16种苏丹卡组合**

#### 类型定义

| 类型 | 代号 | 核心要求 | 设计意图 |
|------|------|----------|----------|
| **纵欲** (Lust) | `LUST` | 寻找床笫之欢 / 与某人共度 | 制造人际关系困境——你的欲望会伤害身边人 |
| **奢靡** (Luxury) | `LUXURY` | 挥金如土 / 消耗大量金钱 | 制造资源困境——奢侈消费挤占生存资源 |
| **征服** (Conquest) | `CONQUEST` | 深入险境 / 侵占土地或人民 | 制造冒险困境——需要军事力量和勇气 |
| **杀戮** (Murder) | `MURDER` | 献祭一条人命 | 制造终极道德困境——谁值得牺牲？ |

#### 等级定义

| 等级 | 代号 | 颜色标识 | 处理难度 | 说明 |
|------|------|----------|----------|------|
| **石** (Stone) | `STONE` | 灰/石色 | 最低 | 基础挑战，新手可处理 |
| **铜** (Bronze) | `BRONZE` | 铜色 | 中低 | 需要一定资源或特定角色 |
| **银** (Silver) | `SILVER` | 银色 | 中高 | 需要稀有资源或多个角色配合 |
| **金** (Gold) | `GOLD` | 金色 | 最高 | 需要关键剧情节点、稀有角色或大量资源 |

> **已验证原版数据**：等级越高，消除路径越少。石卡通常3-5种路径，金卡往往仅1-2种。等级难度递进规律：石→铜→银→金，路径数量递减、资源需求递增、剧情依赖递增。

### 3.2 苏丹卡抽取规则

| 规则 | 说明 |
|------|------|
| **抽取时机** | 每周开始时从牌盒抽取1张 |
| **抽取方式** | 随机抽取（原版规则——顺序完全随机） |
| **处理期限** | 抽取后7天内必须消耗 |
| **逾期后果** | 触发"头身分离术"——玩家直接死亡，游戏结束 |
| **消耗方式** | 通过仪式完成卡牌上对应的挑战 |
| **一次性** | 每张苏丹卡只能使用一次，消耗后消失 |

### 3.3 各类型卡牌的处理路径（已验证原版数据）

> **⚠️ v0.2 重大更新**：以下处理路径基于游民星空/3DM攻略站验证的原版数据，不再使用 PLACEHOLDER。

#### 纵欲卡处理路径

**核心规律**：纵欲卡靠"与角色亲密互动"消除，同品阶优先匹配同品阶角色

| 等级 | 主要处理路径 | 替代路径 | 难度 |
|------|-------------|----------|------|
| 石 | 对石品阶女奴/追随者（快脚、小圆）使用；欢愉之女朱娜 | 随机事件"背叛的朋友/等死的奴隶" | ⭐ 低 |
| 铜 | 对铜品阶王妃安苏亚使用；对铜品阶追随者（铁头、法拉杰）使用 | 梅姬书店密会/浴场约会（各一次）；扎齐伊浴池事件；欢愉之女贾丽拉 | ⭐⭐ 中（路径多） |
| 银 | 对银品阶王妃萨尔达尼使用；对银品阶追随者（奈布哈尼等） | 欢愉之女夏玛；密会对奈费勒；贝姬夫人 | ⭐⭐ 中 |
| 金 | 对金品阶王妃莎姬使用；对金品阶梅姬使用 | 天体派对（夏玛组织） | ⭐⭐⭐ 高（选项少） |

#### 奢靡卡处理路径

**核心规律**：奢靡卡靠"花钱/扩建"消除，金奢靡最依赖长线剧情

| 等级 | 主要处理路径 | 替代路径 | 难度 |
|------|-------------|----------|------|
| 石 | 扩建机制（拖入"俺寻思"消耗3金） | 随机刷新的ji女事件 | ⭐ 低（消耗金钱） |
| 铜 | 扩建机制（消耗5金） | 浴场洗澡→打狗；项目投资；"慷慨解囊"募捐一大笔钱 | ⭐⭐ 中（路径多） |
| 银 | 妻子茶会（循环出现）；出租房屋→买挂毯→扩建 | — | ⭐⭐⭐ 高（需多步准备） |
| 金 | "等重的黄金"→金鸟；扩建金猫雕像 | 合作屠龙线：阿迪莱+哲巴尔结婚 | ⭐⭐⭐⭐ 极高（需屠龙线） |

#### 征服卡处理路径

**核心规律**：征服卡与"战斗/暴力事件"强绑定，铜征服路径最多

| 等级 | 主要处理路径 | 替代路径 | 难度 |
|------|-------------|----------|------|
| 石 | "诸地暴乱"事件中使用 | — | ⭐ 低 |
| 铜 | 登山事件（需一定生存）；"将军助我"；打狗/打狮子；阴森宅邸；硕鼠；被流民侵占宅邸；狼穴恶战 | 7种消除路径，最灵活 | ⭐⭐ 中（路径最多） |
| 银 | 白犀牛事件；盖斯深入调查；远近闻名的哈马斯兄弟 | — | ⭐⭐⭐ 高 |
| 金 | "杀了夏玛她爹！"事件（夏玛剧情后续） | — | ⭐⭐⭐⭐ 极高（唯一路径） |

#### 杀戮卡处理路径

**核心规律**：杀戮卡核心是"俺寻思"消耗角色/事件

| 等级 | 主要处理路径 | 替代路径 | 难度 |
|------|-------------|----------|------|
| 石 | "等死的奴隶"→身体抵债；"欠钱的朋友"→敲闷棍丢入俺寻思 | — | ⭐ 低 |
| 铜 | "被俘的异教徒"；随机杀人事件（俺寻思）；寡妇弑兄 | 硕鼠事件后续 | ⭐⭐ 中 |
| 银 | "带鳄鱼洗澡"事件（需驯兽人+猫获得贵族词条→去浴场） | 组合操作，独特玩法 | ⭐⭐⭐ 高 |
| 金 | "杀宰相"事件（需宰相活着） | — | ⭐⭐⭐⭐ 极高（条件苛刻） |

#### 核心规律总结

| 规律 | 说明 |
|------|------|
| **等级越高，路径越少** | 石卡通常3-5种路径，金卡往往仅1-2种 |
| **金卡最依赖专属剧情** | 金奢靡需屠龙线、金征服需夏玛杀父线、金杀戮需保宰相 |
| **铜卡选择最灵活** | 几乎每种铜卡都有4-8种消除路径 |
| **"俺寻思"是万能入口** | 多种卡牌都可拖入俺寻思触发处理事件 |

## 4. 仪式系统

### 4.1 仪式定义

仪式（Rite）是游戏中的**核心交互单元**——玩家将卡牌配置到仪式中，通过检定获得结果。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 仪式唯一ID（决定结算优先级） |
| `name` | string | 仪式名称 |
| `description` | string | 仪式描述（叙事文本） |
| `location` | enum | 地图位置（宫廷/贫民窟/市场/军营/寺庙/野外） |
| `slots` | list | 配置槽位定义（角色槽/道具槽/苏丹卡槽） |
| `time_limit` | int | 限时天数（红色数字）；0=无限时 |
| `duration` | int | 用时天数（青色数字）；0=立即结算 |
| `required_tags` | list | 槽位需要的卡牌标签 |
| `check_type` | enum | 检定类型（详见检定系统） |
| `outcomes` | dict | 成功/失败/大成功/大失败的结果 |
| `repeatable` | bool | 是否可重复执行 |
| `auto_start` | bool | 是否第2天自动启动（无限时仪式） |

### 4.2 仪式分类

| 类别 | 例子 | 特点 |
|------|------|------|
| **治理仪式** | 治理家业、权力的游戏 | 无限时，每天可执行 |
| **消耗仪式** | 欢愉之馆、赌马 | 限时1天，满足条件时持续重复 |
| **剧情仪式** | 角色专属任务线 | 限时，推动剧情 |
| **苏丹卡仪式** | 各类苏丹卡处理仪式 | 需要放入对应苏丹卡 |
| **探索仪式** | 探索区域、搜寻情报 | 产出资源或触发事件 |

### 4.3 仪式限时规则

| 规则 | 说明 |
|------|------|
| **限时显示** | 地图上仪式旁显示红色数字 = 剩余可用天数 |
| **超时后果** | 通常是不好的发展（负面事件/角色损失），少数有反例 |
| **无限时仪式** | 有小时钟标志，第2天自动启动 |
| **限时1天仪式** | 满足条件会持续重复出现 |

### 4.4 卡牌吸附与锁定

| 规则 | 说明 |
|------|------|
| **吸附顺序** | 按仪式ID从小到大，ID越小优先吸附 |
| **锁定机制** | 被仪式吸附的角色和道具锁定至仪式结束或超时 |
| **策略利用** | 可故意让角色被仪式锁定，避免触发某些需要该角色闲置的事件 |

> **设计假设**：吸附机制增加了策略深度——玩家可以通过角色分配来"规避"某些负面事件。这是预期行为，不是缺陷。

## 5. 检定系统

### 5.1 属性定义（基础八围）

> **⚠️ v0.2 重要修正**：原版有**8个**检定属性（基础八围），不是5个。此外还有5个声望计数器（善名/恶名/权势/侠名/灵视），详见第6章资源系统。

| 属性 | 代号 | 说明 | 典型角色 |
|------|------|------|----------|
| **体魄** | `PHY` | 身体素质、耐力 | 战士、铁头 |
| **战斗** | `COM` | 武力搏斗能力 | 战士、铁头 |
| **生存** | `SUR` | 野外求生、适应力 | 探险者、快脚 |
| **社交** | `SOC` | 人际交往与说服 | 梅姬（妻子）、小圆 |
| **魅力** | `CHA` | 个人吸引力 | 梅姬、夏玛 |
| **隐匿** | `STE` | 潜行与暗中行动 | 刺客、间谍 |
| **智慧** | `WIS` | 知识与谋略 | 学者、梅姬 |
| **魔力** | `MAG` | 魔法力量 | 术士、密教徒 |

> **设计意图**：八围属性的设计让每个角色有明确的"定位"——角色擅长2-3个属性，而非万能。属性值 = 检定时投掷的骰子数，详见5.2。

> `[PLACEHOLDER]`：具体属性值范围需通过测试确定。原版角色属性值通常在2-8之间。

### 5.2 检定流程

> **⚠️ v0.2 核心修正**：原版检定不是"属性+1d6 vs 难度值"，而是**属性值=骰子数、每颗骰子独立判定成功/失败**的二项分布系统。这是与初稿最根本的区别。

```
仪式结算触发检定
     ↓
确定检定属性（仪式定义所需属性及所需成功数）
     ↓
计算骰子总数：
  综合检定：两种属性值相加 = 总骰子数
  分别检定：两种属性分别独立检定
  对抗检定：总骰子数 - 对手属性值 = 实际骰子数
  道具加成：放入对应道具可增加属性值（=增加骰子数）
     ↓
依据本局游戏难度投掷骰子：
  简单难度：每颗骰子成功概率 = 60%
  普通难度：每颗骰子成功概率 = 50%
  困难难度：每颗骰子成功概率 = 33%（游戏描述）或40%（代码数据）
     ↓
统计成功骰子数
     ↓
与检定要求成功数对比：
  ┌─ 大成功（成功数远超要求）
  ├─ 成功（成功数 ≥ 要求）
  ├─ 失败（成功数 < 要求）
  └─ 大失败（成功数远低于要求）
     ↓
可选：使用金骰子（每颗 = +1成功数，检定后追加）
```

> **检定本质**：这是一个**二项分布概率系统**——N颗骰子，每颗有p概率成功，求至少K颗成功的概率。属性值越高→骰子越多→成功概率越高。策略的核心在于：用哪个角色、放什么道具，来最大化骰子数。

### 5.3 骰子机制

> **⚠️ v0.2 重要修正**：金骰子不是"更好的骰子"，而是检定**完成后**使用的追加成功点。

| 类型 | 说明 |
|------|------|
| **普通骰** | 数量 = 参与角色属性值，依据难度决定成功概率 |
| **金骰子** | 检定完成后使用，每颗直接 +1 成功数。前提：骰子总数至少为1（若骰子总数为0则跳过检定，金骰子无法发挥作用） |
| **对抗骰** | 对方属性值从你的骰子总数中扣除 |

> **金骰子获取**：初始 `[PLACEHOLDER: 3]` 枚；可通过特定仪式、剧情事件、命运商店获得。使用需谨慎——金骰子是稀缺的容错资源。

### 5.4 检定难度等级

> **⚠️ v0.2 重要修正**：原版的难度是**全局游戏难度**（开局选择），不是每个仪式独立设定。难度决定所有检定中每颗骰子的成功概率。

| 难度 | 代号 | 单骰成功概率 | 说明 |
|------|------|-------------|------|
| **简单** | `EASY` | 60% | 适合新手体验叙事，可无限回溯 |
| **普通** | `NORMAL` | 50% | 标准体验，策略与运气平衡 |
| **困难** | `HARD` | 33% (描述) 或 40% (代码) | 高难度，需精确规划属性配置 |

> **检定要求成功数**：每个仪式/事件独立设定需要多少颗成功骰子。这个数值才是仪式难度的真正调节器——需要3成功 vs 需要5成功，差别巨大。

> **概率速查**：在普通难度(50%)下，投N颗骰至少K颗成功的概率可用二项分布计算：`P = 1 - BINOM.DIST(K-1, N, 0.5, TRUE)`。此公式可用于构建调优电子表格。

## 6. 资源系统

> **⚠️ v0.2 结构修正**：原版的"资源"包含三个完全不同性质的层级：① 检定属性（八围）= 骰子数来源；② 声望计数器 = 事件触发器；③ 经济资源 = 金币等消耗品。初稿将它们混在一起，现在分离清楚。

### 6.1 三层资源架构

| 层级 | 内容 | 机制 | 用途 |
|------|------|------|------|
| **检定属性层** | 八围（体魄/战斗/生存/社交/魅力/隐匿/智慧/魔力） | 属性值 = 骰子数 | 检定判定 |
| **声望计数器层** | 善名/恶名/权势/侠名/灵视 | 达阈值触发特定事件 | 剧情推进、结局条件 |
| **经济资源层** | 金币、情报卡、食物 | 消耗与获取循环 | 仪式消耗、交易、苏丹卡处理 |

### 6.2 声望计数器详解

> **已验证原版数据**：5种声望有明确的阈值触发机制

| 声望 | 代号 | 核心作用 | 关键阈值 | 获取方式 |
|------|------|----------|----------|----------|
| **善名** | `GOOD_REP` | 正教神迹、角色剧情 | 3（复制神迹）/ 10（复原神迹）/ 20（净化之火） | 正义选择事件、正常剧情推进 |

| **侠名** | `HERO_REP` | 舍馆建设、黑街线 | 3（建舍馆）/ 6（黑街拳斗士）/ 9（罪犯投奔）/ 10（师出有名）/ 20（舍馆升级） | 英勇选择、舍馆经营 |
| **灵视** | `SPIRIT` | 神线专属 | 2（心灵之战）/ 3（幻痛，持有权杖时） | 星象、密教献祭、刺青 |

|------|------|----------|----------|------|

> **已验证原版数据**：奢靡卡的核心处理路径是"扩建"——将奢靡卡拖入"俺寻思"触发

| 等级 | 扩建消耗金币 | 说明 |
|------|-------------|------|
| 石奢靡 | 3 金 | 最基础的奢侈消费 |
| 铜奢靡 | 5 金 | 中等扩建 |
| 银奢靡 | 10 金 | 较大扩建 |

### 6.5 资源产出与消耗循环

```
资源获取（仪式产出）
     ↓
资源储备（玩家库存）
     ↓

```

> **设计意图**：三层资源架构让游戏有清晰的策略空间——属性配置决定检定成功率，声望管理决定剧情走向，金币管理决定生存能力。三者互不替代但相互影响（如权势可稳定获取金币）。

## 7. 事件系统

### 7.1 事件分类

| 类型 | 说明 | 触发条件 |
|------|------|----------|
| **剧情事件** | 推进主线/角色线的故事 | 回合开始自动弹出 |
| **随机事件** | 增加不可预测性 | 每回合随机触发 |
| **限时事件** | 有时间压力的选择 | 限时天数内必须处理 |
| **连锁事件** | 前一事件的结果触发后续 | 事件A的outcome指向事件B |
| **闲置事件** | 需要角色闲置才触发 | 特定角色未被分配到仪式 |

### 7.2 事件数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 事件唯一标识 |
| `trigger_condition` | dict | 触发条件（回合数/角色状态/资源阈值） |
| `text` | string | 叙事文本 |
| `choices` | list | 可选选项列表 |
| `outcomes` | dict | 各选项的结果（资源变化/事件链/角色状态变化） |
| `priority` | int | 事件优先级（决定弹出顺序） |
| `one_time` | bool | 是否只触发一次 |
| `chain_to` | string | 连锁事件的下一个事件ID |

### 7.3 事件触发优先级

| 优先级 | 类型 | 说明 |
|--------|------|------|
| 最高 | 苏丹卡逾期警告 | 死亡倒计时，必须提醒 |
| 高 | 限时事件即将超时 | 制造紧迫感 |
| 中 | 角色闲置事件 | 需要特定角色空闲 |
| 低 | 随机事件 | 增加风味和意外 |

## 9. 叙事与道德系统

### 9.1 叙事风格

- **一千零一夜式**的异域叙事风格

| **异域风格** | 一千零一夜的视觉语言：阿拉伯纹样、暖金色调、手写风格字体 | 沉浸感 |
| **信息层级** | 当前最紧急的信息最显眼（苏丹卡倒计时 > 仪式限时 > 资源） | 紧迫感 |

| **结果戏剧化** | 检定结果不只是数字，而是有情感的故事片段 | 叙事沉浸 |

## 12. 系统交互矩阵与失败状态

### 12.1 系统交互矩阵

> 每对系统的交互标注为：**预期 (E)** / **可接受 (A)** / **缺陷 (B)**

| 系统 A | 系统 B | 交互 | 类型 | 说明 |
|--------|--------|------|------|------|
| 苏丹卡 | 资源 | 奢靡卡消耗金币（扩建3/5/10/20金），征服卡消耗角色/军力 | E | 核心设计意图——卡牌驱动资源压力 |
| 苏丹卡 | 角色 | 纵欲卡与角色亲密互动（消耗角色），杀戮卡献祭角色生命 | E | 道德困境的核心来源 |
| 苏丹卡 | 仪式 | 苏丹卡放入仪式才能消耗 | E | 卡牌-仪式是核心交互方式 |
| 仪式 | 检定 | 仪式触发检定，检定决定仪式结果 | E | 核心反馈链 |
| 仪式 | 资源 | 仪式产出或消耗资源 | E | 资源循环的进出 |
| 仪式 | 角色 | 角色参与仪式，被仪式锁定 | E | 角色分配策略 |
| 事件 | 角色 | 闲置角色触发事件 | E | 制造角色分配困境 |
| 事件 | 资源 | 事件改变资源储备 | E | 事件的经济影响 |
| 资源 | 角色 | 金币可招募角色，资源影响角色属性 | A | 资源-角色的正向循环 |
| 检定 | 角色 | 角色八围属性值 = 骰子数 | E | 角色的核心价值体现 |
| 检定 | 资源 | 道具增加属性值（=增加骰子数） | A | 道具的策略价值 |
| 检定 | 声望 | 声望阈值触发的事件可能改变检定条件 | A | 声望影响检定环境 |
| 苏丹卡 | 事件 | 苏丹卡逾期触发死亡事件 | E | 核心失败条件 |
| 角色 | 角色 | 妻子情绪影响其他角色态度 | A | 角色间的社交传播 |
| 仪式 | 仪式 | 吸附优先级导致角色锁定冲突 | A | 有意的策略深度 |

### 12.2 失败状态定义

> **每个系统必须定义"破坏状态"——知道失败的样子才能识别它**

| 系统 | 破坏状态 | 检测方式 | 恢复手段 |
|------|----------|----------|----------|
| **苏丹卡** | 玩家总有充足资源轻松消耗所有卡牌 | 金级奢靡卡不造成资源危机 | 增加卡牌消耗量 |
| **苏丹卡** | 玩家总是死亡，无法存活超过2周 | 首周死亡率 > 80% | 降低初始卡牌难度 |
| **资源** | 金币无限增长，无消耗压力 | 第3周金币 > 初始值×5 | 增加消耗渠道（扩建、贿赂） |
| **资源** | 金币永远不足，奢靡卡无法处理 | 石级奢靡卡扩建(3金) > 每周稳定收入(5金)→实际可行 | 此项风险降低——3金 < 5金 |
| **声望** | 善名/恶名互斥导致无法触发正教神迹 | 触发善名神迹时恶名≥3 | 降低恶名获取频率或增加善名获取 |
| **声望** | 权势无限增长无代价 | 权势 > 20无负面后果 | 增加权势消耗（猜忌消除需-5权势） |
| **角色** | 角色被永远锁定，无法使用 | 同一角色被锁定 > 3周 | 调整吸附优先级 |
| **情绪** | 妻子情绪永远不满，纵欲卡无路 | 妻子情绪恢复途径 < 1个/周 | 增加情绪恢复事件 |
| **检定** | 随机性过强，策略无意义 | 低属性角色通过高检定的概率 > 50% | 增加属性值差异的权重（骰子数差=概率差） |
| **检定** | 检定永远成功，无紧张感 | 高属性角色成功率 > 95% | 增加检定所需成功数或降低属性上限 |
| **事件** | 事件无影响，玩家不关心 | 事件被忽略 > 50% | 增加事件后果权重 |
| **叙事** | 叙事文本被跳过，不沉浸 | 事件文本阅读时间 < 2秒 | 改善文本质量/排版 |

## 13. MVP 范围与优先级

### 13.1 MVP 定义

**最小可玩版本**：玩家能完成至少一周的完整游戏循环（抽卡 → 7天决策 → 卡牌消耗或死亡），且能感受到核心体验（紧迫感 + 道德困境 + 策略深度）。

### 13.2 MVP 包含清单

| 系统 | MVP范围 | 优先级 |
|------|---------|--------|
| **苏丹卡系统** | 4类型×4等级=16张全实现 | P0 — 核心驱动 |
| **回合管理器** | 每日循环+7天倒计时 | P0 — 核心循环 |
| **仪式系统** | 15个仪式（覆盖每种苏丹卡至少1个处理路径） | P0 — 核心交互 |
| **检定系统** | 八围属性+多骰二项分布检定+金骰子 | P0 — 核心反馈 |
| **资源系统** | 金币+5声望计数器+情报 | P0 — 核心约束 |
| **俺寻思系统** | 卡牌拖入探索触发 | P0 — 核心交互入口 |
| **角色系统** | 5个核心角色（妻子+4盟友） | P0 — 核心载体 |
| **情绪系统** | 妻子情绪值 | P1 — 纵欲卡关键 |
| **事件系统** | 20个关键事件 | P1 — 叙事+意外 |
| **UI框架** | 地图+手牌+仪式配置+检定展示 | P0 — 核心交互 |
| **结局系统** | 4个基本结局 | P1 — 游戏闭环 |

### 13.3 MVP 不包含清单

| 系统 | 原因 | 预计阶段 |
|------|------|----------|
| **跨局成长/积分系统** | 需要先验证单局循环 | Phase 4 |
| **金骰子** | 增加检定复杂度 | Phase 2 扩展 |
| **命运干预** | 特殊机制 | Phase 4 |
| **随机事件池** | MVP用固定事件池 | Phase 3 |
| **创意工坊** | 非学习目标 | 不做 |
| **音效/配乐** | 降低初期复杂度 | Phase 4 可选 |
| **多语言** | 降低工作量 | 不做 |
| **存档系统** | MVP不需要持久化 | Phase 3 |

### 13.4 MVP 验收标准

| 标准 | 测试方法 |
|------|----------|
| 完整一周循环可玩 | 从抽卡到消耗/死亡，流程无断点 |
| 玩家感受到紧迫感 | 新手首次游玩在3周内死亡 > 60% |
| 玩家做出道德困境选择 | 纵欲/杀戮卡至少有2种处理路径 |
| 策略有意义 | 不同卡牌配置导致不同的检定成功率 |
| UI可操作 | 拖拽卡牌到仪式能正确配置和结算 |
