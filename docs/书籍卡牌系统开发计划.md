## 原版机制速查

### 获取书籍：古籍摊

| 项目 | 说明 |
|------|------|
| 触发 | 点击地图上的「古籍摊」（常驻仪式 id=16，与玫瑰庄园是不同仪式） |
| 消耗 | 拖入金币卡（1金币） |
| 产出 | 随机一本**未拥有过**的书（卡池约 60 本） |
| 限次 | 购买 20 次后书店关闭，触发「淘书」事件链 |
| 超时 | 不限时 |

### 读书：拖入灵光一现

| 项目 | 说明 |
|------|------|
| 触发 | 书籍卡拖入灵光一现 → 生成「看书」仪式 |
| 限时 | 1 天，超时书籍回收 |
| 耗时 | 1 天 |
| 角色 | 需要 1 名角色执行，有**智慧/魔力门槛**（鲁梅拉无视） |
| 结果 | 角色对应属性 +N，书消失（部分冒险书不消失但仅首次加属性） |

### 品级系统

书籍和情报/卡牌共用四品级（石/铜/银/金），品级越高属性增益越大、门槛越高。

---

## MVP 范围

- **书籍数量**：6 本（石×1 / 铜×2 / 银×2 / 金×1）
- **获取**：点击「古籍摊」 → 拖入金币卡 + 可选角色 → 获得随机书籍卡
- **读书**：拖书到灵光一现 → 生成看书仪式 → 角色加属性
- **不实现**：淘书事件链、冒险书、焚书路线、借书证

**字段说明**：

| 字段 | 含义 |
|------|------|

| attr | 增益属性：social / combat / wisdom / charm / stealth / magic |
| gain | 属性增加值（金品级可 +2） |
| req | 所需智慧最小值，0 为无门槛 |

**不改动文件**：`rites.json` / `characters.json` / `sultan_cards.json` / `events.json`

### Step 4 — 古籍摊仪式发书

**状态**：✅ 仪式数据已就绪（id=16，常驻，slots: 金币卡(必填)+角色(可选)）

**改动文件**：`scripts/ui/MainScene.gd`

在古籍摊按钮的 `pressed` 回调中：

1. 过滤 `DataManager.books`，排除已有书籍
2. 随机选一本，扣 1 金币
3. 调用 `_add_card_to_hand()` 或等效方法把书籍卡加到手上
4. 日志：`_log("📖 购得《xxx》")`

---

### Step 5 — 灵光一现读书

**改动文件**：`scripts/ui/MainScene.gd`

在 `_do_insight_with_card()` 中新增分支：

```
拖入卡牌 type == "book"
  → 检查本轮已用（同类仅一次）
  → 调用 _find_insight_rites("book", book_data)
  → 返回匹配的看书仪式（insight_trigger, 限时1天）
  → 消耗书籍卡
```

需要新增的仪式数据（在 `data/rites.json` 中追加，或代码中动态生成）：

```json
{
  "id": 300,
  "name": "看书",
  "category": "insight",
  "time_limit": 1,
  "insight_trigger": {"type": "book", "subtype": "READ"},
  "duration": 1,
  "slots": [{"type": "character", "required": true}]
}
```

**关键**：「看书」仪式结算时，读取书籍的 `attr` 和 `gain`，增加角色属性。门槛检查在结算时进行。
