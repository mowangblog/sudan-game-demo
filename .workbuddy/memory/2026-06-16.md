## Bug 修复 — 手写 .tscn 导致 null 实例
- 问题：手写的 .tscn 格式不精确，Godot 解析时部分子节点未创建，$路径返回null
- 解决方案：**彻底放弃 .tscn 子节点，改为纯代码构建所有 UI**
- MainScene.tscn 只保留根 Control 节点 + MainScene.gd 脚本引用
- MainScene.gd 在 _ready() 中用代码创建所有 UI（StatusBar/MapArea/InfoPanel/HandArea/NextDayButton）
- 删除不再使用的独立 UI 脚本（StatusBar.gd/MapArea.gd/HandArea.gd/InfoPanel.gd）
- 修复 GameManager._on_day_started 的状态保护（INIT 状态忽略信号）
- 修复 TurnManager 时序：第1天在 _ready 的 call_deferred 中发出
- 修复 project.godot 的渲染配置（去掉 d3d12，改为 forward_plus）

### 经验教训
- **手写 Godot .tscn 非常脆弱**，格式稍有偏差节点就创建不出来
- Phase 1 项目应优先使用纯代码构建 UI，等稳定后再用编辑器生成 .tscn
- AutoLoad 的 _ready 顺序按注册顺序执行，需要注意信号时序

### 当前可玩流程
1. 启动 → 自动抽卡（显示在中央）→ 日志显示"苏丹降下命令"
2. 点击卡牌 → 金色高亮 → 日志提示"选中XX，请点击地点"
3. 点击地点按钮 → 弹窗列出可用的仪式（按卡类型筛选）
4. 选择仪式 → 骰子滚动动画 → 显示通过/失败
5. 成功 → 卡牌消失 → 自动进入下一周 → 新卡
6. 失败 → 卡牌保留 → 继续尝试或点"下一天"
7. 点击"下一天" → 苏丹卡倒计时-1 → 7天后逾期 → GAME OVER
